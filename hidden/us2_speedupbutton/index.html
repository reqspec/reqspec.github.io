US-2 Настройка прироста веса Сделать приоритетным

Описание
Обоснование критичности
Требования

Описание:

Как нач. группы 1 или 2 линии
Я хочу Настроить прирост веса по событию “Ускорить обращение” на сумму значение веса самого приоритетного обращения на роли и текущего веса обращения
Для быстрого редактирования прироста веса

Критичность : КРИТИЧНО
Источник информации : Бизнес-анализ
Фаза: Фаза 1

Обоснование критичности

Без данной US невозможно выполнение целевой US-1.

Требования

Этап: 1 фаза
Названиe: БТ-3 Добавить событие “Ускорить обращение” в “Центр приоритизации”
Система: Отклик
Описание: Добавить в список событий (выпадающий список в модальном окне после нажатия кнопки “Добавить”) событие “Ускорить обращение”.

Событие соответствует нажатию кнопки “Ускорить обращение” (Подробнее см. US-1).

Источник: Бизнес-анализ
Критичность: КРИТИЧНО

Этап: 1 фаза
Названиe: БТ-4 Добавить возможность указания типа прироста веса к обращению
Система: Отклик
Описание: Добавить новое поле с выпадающим списком в Модальном окне “Событие” с названием “Тип прироста” Значения для поля:

  • Сделать приоритетным

Поле обязательное для заполнения. Подробное описание в на странице UI-2 (UI-2 ЭФ Окно “Событие”)

Источник: Бизнес-анализ
Критичность: КРИТИЧНО

Этап: 1 фаза
Названиe: БТ-5 Логика типа прироста “Сделать приоритетным”
Система: Отклик
Описание:
ЕСЛИ для события указан тип прироста “Сделать приоритетным” И настройка по событию сохранена
ТО при наступлении события вес обращения становится равному значению: сумма значения текущего веса обращения и самого большого веса среди обращений, находящихся на идентичной роли.
Формула: A=B+C

Где:
A - новый вес обращения
B - наибольшее значение веса среди обращений, находящихся на идентичной роли
C - Текущий вес обращения

Источник: Бизнес-анализ
Критичность: КРИТИЧНО
